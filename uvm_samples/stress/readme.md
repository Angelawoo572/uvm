The directories in here are supposed to be stress tests for UVM generation. With the exception of the first example which is only concerned with building basic connectivity in a MWE fashion, the others stress one component at a time.
The DUT may change from one example to another.

example1-basic is just a simple instantiation of all typical UVM components. It is a sanity check of sorts.
example2-reqrspdriver uses the REQ/RSP style for the driver/sequencer/sequence handshake. The DUT is a bit more interesting for this one.
example3-cov_collector attemps to exercise different styles of coverage collection. It creates cgs in different ways, as well as bins and crosses.
